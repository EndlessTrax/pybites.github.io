How to Debug a Hanging Test Using the pytest-timeout Plugin

Posted by Bob on Wed 18 March 2020 in Testing • 3 min read

Today a neat trick that might save you quite some headache: debugging a hanging test.

Setup

Let's write some really simple code to test:

from time import sleep

def call_api():
    return dict(
        status=200,
        response=[1, 2, 3])


def sum_numbers(numbers):
    return sum(numbers)


def parse_website():
    resp = call_api()
    # more parsing but just need a function
    return resp

And some test code to test it:

from script import sum_numbers, call_api


def test_sum_numbers():
    assert sum_numbers([1, 2, 3]) == 6
    assert sum_numbers([4, 5]) == 9


def test_call_api():
    resp = call_api()
    assert resp['status'] == 200
    assert resp['response'] == [1, 2, 3]

Run pytest and: 2 passed in 0.03s - all good.

Now let's emulate a hanging test by adding a sleep of 60 seconds in call\_api

Eventually you'll get there (2 passed in 60.06s (0:01:00)), but this might be far worse and even with 1 or 2 minutes delay, you need to fix it, because tests need to be fast as you will constantly run them to refactor and add new feature.

Debugging timeouts

I discovered a neat plugin for this: pytest-timeout. With your virtualenv enabled:

pip install pytest-timeout

Let's use it:

(venv) [bobbelderbos@imac test-debug]$ pytest --timeout=3

A more complex use case

What if the bug is in some external module?

Here I pulled a fork of requests and:

  • added a import time; time.sleep(60) to the get function of the api module,

  • added requests.get('https://pybit.es') to my call\_api function,

  • cd'd into the requests subfolder and did a pip install -e . to install THAT modified version (as opposed the one on the net)

And ... boom!

What if the failing code is more layers down?

Sometimes the issue is a more nested, lot of functions calling each other and pytest-timeout might not be that smart.

In that case you can swap the default signal of the --timeout_method flag for thread and it will dump a complete stacktrace!

This can be very useful for debugging:

(venv) [bobbelderbos@imac test-debug]$ pytest test_script.py --timeout=3 --timeout_method=thread
